# Contribution Guidelines

Thank you for helping grow this list! Please read these guidelines before submitting.

## Criteria

To be added to the list, a project must meet **all** of the following:

1. **Public repository** — The source code is hosted on a public GitHub (or similar) repository.
2. **Has a README** — The repo contains a README that describes what the project does.
3. **Built with AI / vibe coding** — The majority of the codebase was generated through AI-assisted development (e.g., Claude, ChatGPT, Cursor, GitHub Copilot, Windsurf, Bolt, v0, Lovable, Replit Agent, etc.).
4. **Working or has a demo** — The project either runs, is deployed, or has a video/screenshot demo showing it in action.

## How to Submit

### Option A: Pull Request

1. Fork this repository.
2. Add your project to the appropriate category in `README.md`.
3. Use the following format:

   ```
   - [project-name](https://github.com/user/repo) - Short description of what it does. `technology` `ai-tool`
   ```

4. Keep the description concise — one sentence, no marketing fluff.
5. Place the entry in **alphabetical order** within its category.
6. Submit a pull request with a clear title, e.g., `Add project-name to SEO & Digital Marketing`.

### Option B: GitHub Issue

1. Go to [Issues](../../issues/new/choose).
2. Select the **Add Project** template.
3. Fill in the fields and submit.
4. A maintainer will review and add it to the list.

## Quality Standards

- The description must accurately reflect what the project does.
- No affiliate links or paid promotions.
- One project per pull request.
- If the project is no longer maintained or the repo is archived, it may be removed.

## Updating an Entry

If a project URL changed or the description needs updating, submit a PR with the correction.

## Categories

If your project does not fit any existing category, suggest a new one in your PR description. New categories are added when there are at least 2-3 projects that fit.
